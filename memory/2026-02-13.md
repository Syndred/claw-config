## 学习/收获

- 流程设计要考虑依赖顺序
- 记忆文档是旺旺的"大脑备份"，必须及时同步
- **日记时间格式**: 主题不会自动转换时区！要写准确的时间（今天的事是11:15发生的，不是凌晨3:15）
- **包管理器**: 项目使用 `pnpm`，不是 `npm`（配置了only-allow pnpm）
- **先本地测试**: 必须 `pnpm build` 通过后再提交，不能偷懒
- **中文引号**: `"文字"` 会导致esbuild解析失败，要用 `「文字」`

## 教训总结

**以后写日记的 checklist:**
- [ ] 时间写准确（几点几分就写几点几分，UTC时间）
- [ ] 检查中文引号，改成直角引号
- [ ] 本地 `pnpm build` 通过
- [ ] 再提交推送

**记住的事情必须写进记忆文档！这是防止失忆的唯一方法。**

## OpenClaw更新教训

**问题**: 用`npm i -g`升级失败，一直卡住没反应  
**原因**: OpenClaw实际是用`yarn global`安装的，不是npm  
**解决**: `yarn global upgrade openclaw@latest`  
**下次升级前**: 先检查 `openclaw status` 看channel，或用 `which openclaw` 看路径

```bash
# 正确的升级方式
yarn global upgrade openclaw@latest
```
